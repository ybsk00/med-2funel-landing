11개 진료과별 디자인 및 기능 구현 상세 계획서
본 계획서는 
department_design_proposal.md
에 정의된 11개 진료과의 테마, 인터랙션, 기능을 실제 코드로 구현하기 위한 단계별 상세 로드맵입니다.

🏗 Phase 1: 기반 아키텍처 및 라우팅 (Foundation) - [진행 중]
목표: 11개 진료과 페이지 생성 및 기본 데이터 구조 연결

Step 1.1: 컨피그레이션 고도화 (Configuration Refactor)
 
src/lib/constants/departments.ts
 데이터 구조 확장
 designTheme 객체 추가: texture (재질), font (폰트), sound (사운드 파일 경로)
 copywriting 객체 추가: catchphrase, subtext
 11개 과별 신규 네이밍 및 비디오 경로 매핑 (/히어로세션/...)
Step 1.2: 정적 라우트 및 페이지 생성
 주요 진료과 (/dermatology, /dentistry 등) 페이지 파일 생성 완료
 
GenericDepartmentPage
를 대체할 각 과별 전용 컴포넌트 스캐폴딩 생성
 예: src/components/departments/PlasticSurgeryLanding.tsx
 예: src/components/departments/KoreanMedicineLanding.tsx
 (이하 11개 과 모두 별도 컴포넌트화 하여 커스텀 디자인 허용)
Step 1.3: 에셋 준비
 public/sounds/ 디렉토리 생성 및 사운드 효과(크리스탈, 대나무, 엔진음 등) 플레이스홀더 파일 생성.
🎨 Phase 2: 시각적 테마 및 카드 디자인 구현 (Visual Design)
목표: 각 과별 독창적인 "공간감"과 "카드 스타일" 구현

Step 2.1: 테마 엔진 고도화
 
HospitalProvider
에 Texture 및 Font 클래스 주입 로직 추가.
 Tailwind CSS 커스텀 클래스 정의 (예: .bg-texture-hanji, .bg-texture-carbon)
Step 2.2: 진료과별 전용 카드(Card) 컴포넌트 개발
 src/components/ui/cards/ 내에 과별 카드 컴포넌트 개발:
 VipCard.tsx (성형외과 - 블랙/골드, 금가루 효과)
 HanjiCard.tsx (한의원 - 한지 질감, 세로쓰기 지원)
 GlassCard.tsx (치과 - 투명 글래스모피즘, 네온 발광)
 BlueprintCard.tsx (정형외과 - 설계도면 스타일)
 SportsCard.tsx (비뇨기과 - 카본 텍스처, 사선 디자인)
 JellyCard.tsx (소아과 - 모서리 둥글고 눌리는 효과)
 FlowerCard.tsx (산부인과 - 꽃잎 레이어)
 BotanicCard.tsx (내과 - 식물 도감 스타일)
 LinenCard.tsx (암요양 - 리넨 질감)
 HologramCard.tsx (피부과/신경외과 - 홀로그램/칩 스타일)
Step 2.3: 레이아웃 시스템 구현
 Horizontal Scroll Container: 한의원 병풍 레이아웃용.
 Masonry Grid: 암요양 벽돌 쌓기 레이아웃용.
 Floating Bubbles: 소아과 버블 레이아웃용 (물리 엔진 적용).
 Bento Grid: 정형외과 벤토 박스 레이아웃용.
⚡ Phase 3: 인터랙티브 모듈 개발 (Interaction Implementation)
목표: 각 과의 핵심 기능을 시각적으로 체험하는 "킬러 콘텐츠" 구현

Step 3.1: 시뮬레이션 모듈 (Priority A)
 가상성형 AI 미러 (성형외과)
 이미지 업로드 및 react-compare-slider 커스텀 구현 (마스킹 드래그).
 황금비율 가이드라인 오버레이 (SVG).
 스노우 화이트 미백 (치과)
 치아 이미지 마스킹 및 filter: brightness() 조절 슬라이더.
 CSS backdrop-filter를 활용한 광택(Glossy) 효과.
 슈퍼 성장 예측 (소아과)
 Framer Motion SVG 애니메이션: 기린 목이 늘어나는 모션.
Step 3.2: 진단 및 측정 모듈 (Priority B)
 내 몸의 기운 스캐너 (한의원)
 Canvas API 또는 SVG Filter를 활용한 먹물 번짐(Ink Splash) 효과 구현.
 체형 밸런스 각도기 (정형외과)
 카메라/이미지 위에 Draggable 각도기 컴포넌트 오버레이.
 각도 측정 시 햅틱 피드백 연동.
 활력 RPG 게이지 (비뇨기과)
 Canvas Gauge 라이브러리 활용하여 RPM 스타일 게이지 구현.
Step 3.3: 3D 및 이펙트 모듈 (Priority C)
 두통 3D 매핑 (신경외과)
 Three.js (또는 React Three Fiber) 활용 두뇌 모델 렌더링.
 클릭 부위 발광(Emissive) 효과.
 이뮤니티 쉴드 (암요양)
 여러 겹의 SVG 원형 파동이 퍼지는 글로우 효과.
🔒 Phase 4: 감각 경험 및 보안 (Sensory & Security)
목표: 청각, 촉각, 심리적 안정감을 주는 디테일 보강

Step 4.1: 사운드 시스템 (Audio Engine)
 useSoundEffect 훅 개발: 클릭, 호버, 완료 시 사운드 재생.
 각 페이지 마운트 시 과별 전용 사운드 에셋 프리로드.
Step 4.2: 햅틱 피드백 (Haptic)
 navigator.vibrate API 래퍼 함수 개발.
 모바일 환경 감지 후 햅틱 패턴 적용 (약한 진동, 강한 진동, 패턴 진동).
Step 4.3: 보안 애니메이션 (Data Anonymization)
 공통 Submit 버튼 컴포넌트 개발 (SecureSubmitButton).
 클릭 시 텍스트가 Matrix 문자열(#@!$)로 변환되는 애니메이션 구현.
 "데이터 암호화 전송 중" 토스트 메시지와 자물쇠 아이콘 애니메이션.
🗓 우선순위별 개발 순서
데이터 & 라우팅 (Step 1.1 ~ 1.2): 11개 과 페이지가 모두 정상적으로 떠야 함.
카드 디자인 (Step 2.2): 각 과 페이지에 들어갔을 때 테마가 느껴져야 함.
핵심 인터랙션 (Step 3.1): 성형외과, 치과, 소아과 등 시각적 효과가 큰 모듈부터 선행 개발.
보안 및 사운드 (Phase 4): 전체적인 퀄리티 업그레이드.
