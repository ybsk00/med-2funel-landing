# Markdown 설정 파일 → YAML 변환 가이드

## 방법 1: 수동 변환 (5분)

입력한 Markdown 파일을 참고하여 `config/hospital-config.yaml` 파일을 직접 작성합니다.

## 방법 2: 자동 변환 스크립트 (권장)

### Step 1: 변환 스크립트 실행

```bash
# 개발 서버에서
npm run config:convert 병원설정입력양식.md
```

### Step 2: 자동 생성 확인

`config/hospital-config.yaml` 파일이 자동 생성됩니다.

### Step 3: 배포

```bash
npm run build
npm run deploy
```

## 입력 → 변환 흐름도

```
┌─────────────────┐
│ 병원설정입력양식.md │  ← 작성
└────────┬────────┘
         ↓
┌─────────────────┐
│  파싱 & 변환     │  ← 자동
└────────┬────────┘
         ↓
┌─────────────────┐
│hospital-config.yaml│ ← 생성
└────────┬────────┘
         ↓
┌─────────────────┐
│     빌드        │  ← npm run build
└────────┬────────┘
         ↓
┌─────────────────┐
│     배포        │  ← 사이트 오픈!
└─────────────────┘
```

## 각 과목별 자동 적용 내역

과목만 선택하면 아래 내용이 **자동으로 설정**됩니다:

| 항목 | 자동 적용 내용 |
|------|---------------|
| 헬스케어 모듈 5개 | 과목별 맞춤 (피부과→광채/장벽 등) |
| AI 페르소나 톤 | 과목별 최적화된 톤 |
| 상담 트랙 6~8개 | 과목별 전문 키워드 |
| 색상 테마 | 과목별 시그니처 색상 |
| 아이콘 세트 | Lucide 아이콘 자동 매핑 |

**커스터마이징이 필요한 경우에만** 상세 입력 양식을 사용하세요.

## 파일 위치

```
프로젝트 루트/
├── 병원설정입력양식.md      ← 입력 파일
├── 병원설정_간단입력.md      ← 간단 버전
├── config/
│   └── hospital-config.yaml  ← 생성된 설정
└── src/modules/config/defaults/  ← 과목별 기본값
    ├── dermatology.ts
    ├── dentistry.ts
    └── ...
```
