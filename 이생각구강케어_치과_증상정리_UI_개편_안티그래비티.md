# 이생각 구강 케어 — 「치과 증상정리」(Symptoms Triage) UI 개편 작업지시서 (안티그래비티)

목적: 기존 **증상정리(소화/통증재활/면역/여성건강/인지·수면)** 구조를 **치과/구강 중심**으로 전환하고, 사용자가 1~2분 내에 **“내 구강 불편 상황을 정리”**하여 상담 준비용 요약을 만들도록 한다.  
주의: 본 플로우는 **진단/치료/효과 보장**을 하지 않는다. “의료정보 제공 + 상담 준비용 정리” 톤 유지.

---

## 1) 화면 네이밍/카피(의료법 저위험 톤)

### 모달 타이틀(좌상단)
- 기존: `증상 정리`
- 변경(추천): `구강 불편 정리`  
  - 대안: `구강 컨디션 정리(참고용)` / `구강 체크(참고용)`

### 서브 카피(타이틀 아래 작은 문구)
- `간단한 선택으로 현재 상태를 정리합니다. 결과는 상담 준비용 참고이며 진단이 아닙니다.`

### 하단 고정 고지(필수)
- `본 체크는 의료정보 제공 및 상담 준비용 참고입니다. 증상이 지속·악화되면 의료진 상담이 필요합니다.`
- 응급/레드플래그 상황 추가(작은 링크형): `응급 징후 보기` (클릭 시 안내 텍스트 펼침)

---

## 2) Step 구조(1/6 유지, 내용 치과화)

> 기존 스텝 수(1/6)는 유지하되, 각 스텝의 질문/옵션을 치과/구강에 맞게 변경한다.  
> 스텝은 **선택형 → 짧은 입력 → 요약**으로 구성해 이탈을 줄인다.

### Step 1/6: “어떤 불편이 가장 크신가요?” (카테고리 선택)
**6개 카드(현 레이아웃 유지) 권장**

1. **치아 통증/시림**
   - 설명(작게): `찌릿함·깨물 때 통증`
2. **잇몸 불편**
   - 설명: `붓기·피·통증`
3. **치아 손상/충치 의심**
   - 설명: `파절·구멍·변색`
4. **턱관절/이갈이**
   - 설명: `턱 통증·딱딱 소리`
5. **구취/구강건조**
   - 설명: `입냄새·마름`
6. **교정/보철·임플란트**
   - 설명: `교정장치·크라운·임플란트 관련`
- 기타는 **6개 카드 안에서 “교정/보철…” 카드 내부 옵션**으로 흡수(UX 단순화).  
  (정말 필요 시 Step 1 하단에 작은 링크: `기타 증상 선택`)

**아이콘 가이드(추상/중립)**
- 실사 치아/잇몸 사진 금지(광고/의료행위 연상 최소화)
- 라인 아이콘(치아/잇몸/턱/물방울/브라켓 등) 사용

---

### Step 2/6: “어디가 불편하신가요?” (부위 선택)
- 선택 방식: **치아맵(간단) + 리스트 병행**
- 옵션(예시):
  - `앞니` `어금니` `잇몸` `턱관절` `전체` `잘 모르겠음`
- 치아맵은 고도화하지 말고 MVP는 리스트로도 충분.

---

### Step 3/6: “언제부터 / 얼마나 자주인가요?” (기간/빈도)
- 기간: `오늘` `2~3일` `1주 이상` `1개월 이상` `반복됨`
- 빈도: `가끔` `하루 1~2회` `매일` `지속됨`

---

### Step 4/6: “언제 더 불편해지나요?” (트리거/상황)
- 공통 옵션:
  - `차가운/뜨거운 음식` `단 음식` `씹을 때` `가만히 있어도` `양치할 때` `아침에 심함` `밤에 심함` `잘 모르겠음`
- Step 1 카테고리에 따라 **조건부 옵션 추가**
  - 잇몸 불편: `양치/치실 시 피` `붓기` `고름 느낌`
  - 턱관절: `하품` `입 벌릴 때` `딱/걸림`

---

### Step 5/6: “불편 정도를 선택해주세요.” (강도/동반 증상 + 레드플래그 체크)
- 강도(1~5): `가벼움(1)` ~ `매우 심함(5)`
- 동반 증상 체크(멀티):
  - `붓기` `출혈` `열감` `입 벌리기 어려움` `삼키기 어려움` `얼굴 비대칭` `고름/악취` `발열`
- **레드플래그(즉시 안내 문구)**
  - `호흡 곤란, 삼킴 곤란, 얼굴이 급격히 붓는 경우, 고열 동반` 선택 시:
    - 안내: `응급 상황일 수 있어 즉시 의료기관/응급실 상담이 필요합니다.`
    - (여기서는 병원 유도 CTA 금지, “안전 안내”만)

---

### Step 6/6: 결과 요약(상담 준비용)
**출력 구성(카드형 3줄 요약 + 상세)**
- 한 줄 제목: `내 구강 불편 요약(참고용)`
- 핵심 3줄(자동 생성):
  1) `주요 불편:` {카테고리/부위}
  2) `기간/빈도:` {기간}/{빈도}
  3) `유발 상황:` {트리거} / `강도:` {1~5}
- 상세 리스트(선택): 동반 증상, 추가 메모(사용자 입력)

**버튼(저위험 전환 설계)**
- 1순위: `요약 저장(로그인)`  
- 2순위: `다른 증상도 정리하기`
- *주의:* “예약/상담 연결” 같은 직접 CTA는 헬스케어 화면에서 최소화. 저장/기록 목적의 로그인 유도가 더 안전.

---

## 3) “운영정보 보기 → 로그인 → 안내 카드”와의 연결 방식(권장)

- 증상정리 완료(6/6)에서 **로그인 유도 버튼을 “저장” 목적**으로 노출한다.
- 로그인 후 이동 화면(메디컬/채팅) 상단에는:
  - **(1) 선택한 요약 카드**
  - **(2) 병원 운영정보 카드(중립 톤)**  
  를 **동시에 상단 고정**하여 사용자가 “왜 로그인했는지” 즉시 이해하도록 한다.

**로그인 유도 모달 문구(짧게)**
- 제목: `요약 저장`
- 본문: `결과를 저장하고 나중에 다시 보려면 로그인이 필요합니다.`
- 버튼: `로그인하고 저장` / `닫기`

---

## 4) 데이터 모델(프론트/백 공통)

### 필드 제안(예시)
```json
{
  "category": "tooth_pain",
  "area": ["molar"],
  "duration": "1_week_plus",
  "frequency": "daily",
  "triggers": ["chewing", "cold"],
  "severity": 4,
  "symptoms": ["swelling"],
  "red_flags": [],
  "memo": "오른쪽 어금니 씹을 때 통증",
  "created_at": "ISO-8601"
}
```

### 저장 정책
- 비로그인: LocalStorage(세션) + 결과 화면 표시
- 로그인: DB 저장(사용자별) + 채팅 상단 카드로 주입

## 6) 문구 가이드(금지/권장)

### 금지(리스크 높음)
- `진단`, `치료`, `완치`, `효과`, `보장`, `전문의가 판단` 등 확정 표현
- `충치입니다`, `잇몸염입니다` 등 질환 단정
- 전/후 비교, 시술 장면, 의료기기 강조

### 권장(저위험)
- `정리`, `참고`, `패턴`, `상담 준비`, `확인`
- `변동될 수 있음`, `의료진 상담 권장`

---

## 7) QA 시나리오(필수 테스트)

1) Step 1~6 정상 진행 및 뒤로가기 동작 확인
2) 레드플래그 선택 시 안전 안내 출력 확인(CTA 없음)
3) 비로그인 상태에서 “요약 저장” → 로그인 이동(returnUrl) 확인
4) 로그인 성공 후 returnUrl 복귀 + 요약 카드 자동 표시 확인
5) 데이터 저장/조회(사용자별) 확인
